# 产品指标仪表盘设计

---

## 1. 设计目标

为B端企业管理者（VP / 培训经理 / 区域主管）提供**培训效果可量化、团队短板可定位、个人表现可追踪**的数据化决策工具。仪表盘是产品从"培训工具"升级为"管理平台"的核心价值承载页面。

| 设计原则 | 说明 |
|---------|------|
| 3秒可读 | 概览卡片让管理者一眼看到核心数字 |
| 10秒定位 | 热力图/排行榜快速定位问题团队和个人 |
| 1分钟决策 | 趋势图+异常提醒辅助管理者做出培训资源调配决策 |

---

## 2. 角色视图差异

| 角色 | 数据范围 | 默认视图 | 核心关注点 |
|------|---------|---------|-----------|
| **销售VP/总监** | 全公司 | 全公司概览卡片 + 部门对比柱状图 | ROI趋势、区域差距、整体达标率 |
| **培训经理** | 全公司（个人数据脱敏） | 异常提醒 + 维度短板热力图 | 哪些团队/个人需要介入、培训任务完成率 |
| **区域主管** | 本区域数据 | 本区域个人排行榜 + 趋势图 | 谁表现好/差、区域整体是否在进步 |

权限控制基于RBAC：超管和培训经理可切换查看全局/任意区域数据，区域主管仅限本区域，学员仅限个人数据。

---

## 3. 仪表盘模块详细设计

### 3.1 概览卡片（4个核心数字）

| 卡片 | 数据口径 | 刷新频率 | 交互动作 |
|------|---------|---------|---------|
| **总学员数** | 当前企业内已激活学员总数（排除已离职/禁用账号） | 实时 | 点击进入学员列表页 |
| **本月陪练次数** | 当前自然月所有学员完成的陪练会话总数（状态=completed） | 每小时 | 点击展开按周分布柱状图 |
| **平均得分** | 本月所有已完成陪练的评估得分均值（5维加权总分） | 每小时 | 点击展开按维度分解雷达图 |
| **本周活跃率** | 本周至少完成1次陪练的学员数 / 总学员数 | 每日 | 点击查看近4周活跃率变化趋势 |

**展示形式**：4个卡片横向排列，大数字 + 环比变化百分比（↑/↓箭头 + 颜色：绿涨红跌灰平）。卡片下方附一行小字说明数据截止时间。

---

### 3.2 部门对比柱状图

| 属性 | 说明 |
|------|------|
| **展示形式** | 双轴柱状图：X轴=部门/区域/门店，Y1轴=陪练次数（柱状），Y2轴=平均得分（折线叠加） |
| **数据口径** | 当月数据，按用户所属组织架构group by |
| **排序** | 默认按平均得分降序排列 |
| **粒度切换** | 支持按区域→门店→小组三级下钻 |
| **刷新频率** | 每小时 |
| **交互动作** | 点击某个部门柱状→进入该部门独立看板；右上角切换时间范围（本月/本季/本年） |
| **权限控制** | VP/培训经理：全部部门；区域主管：仅本区域及下级部门 |

---

### 3.3 维度短板热力图

| 属性 | 说明 |
|------|------|
| **展示形式** | 热力图矩阵：X轴=5大维度（沟通表达/需求挖掘/产品介绍/异议处理/成交推进），Y轴=部门/区域，颜色=得分（红=薄弱 <60，黄=中等 60-75，绿=优秀 >75） |
| **数据口径** | 近30天内各部门在各维度的平均得分 |
| **刷新频率** | 每日 |
| **交互动作** | 悬停显示具体分数和参评人次；点击某个格子→跳转到该部门该维度的详细分析页（展示该维度的子评分点分解） |
| **预警阈值** | 红色单元格自动触发标注，培训经理登录时优先展示 |
| **权限控制** | VP/培训经理：全公司热力图；区域主管：仅本区域行 |

---

### 3.4 个人排行榜（Top10 + Bottom10）

| 属性 | 说明 |
|------|------|
| **展示形式** | 左右双列排行：左=Top10优秀学员（近30天平均得分），右=Bottom10需关注学员（近30天平均得分）。每人展示：排名、姓名、部门、平均分、陪练次数、得分趋势箭头 |
| **数据口径** | 近30天至少完成3次陪练的学员才进入排名（避免样本过少） |
| **刷新频率** | 每日 |
| **交互动作** | 点击学员姓名→进入该学员个人详情页（能力雷达图+成长曲线+陪练历史） |
| **延伸功能** | 支持按部门/入职批次筛选，支持导出名单（Excel） |
| **敏感处理** | Bottom10在培训经理视图中直接展示（用于辅导）；VP视图仅展示Top10，Bottom10折叠为"需关注学员（N人）"，点击展开（避免制造不必要的压力） |
| **权限控制** | VP：Top10直接展示 + Bottom10折叠；培训经理：全部展示；区域主管：仅本区域学员 |

---

### 3.5 趋势图（12周）

| 属性 | 说明 |
|------|------|
| **展示形式** | 折线图：X轴=周（近12周），Y轴=团队平均得分。可叠加展示陪练总次数的面积图作为辅助 |
| **数据口径** | 每周所有完成陪练的评估得分均值 |
| **刷新频率** | 每周一更新上周数据 |
| **交互动作** | 切换对比维度：按部门对比（多条折线）、按难度对比、按场景对比；可拖拽选择时间范围缩放 |
| **标注** | 当周平均分出现显著变化（↑↓超过10分）时自动在图上标注可能原因（如"本周上线新场景模板""本周新人入职批量陪练"） |
| **权限控制** | 数据范围与角色权限一致 |

---

### 3.6 异常提醒

| 提醒类型 | 触发条件 | 展示形式 | 交互动作 |
|---------|---------|---------|---------|
| **连续下降** | 某学员连续3次陪练得分下降 | 列表项 + 红色警示图标 | 点击进入该学员详情，查看具体下降维度 |
| **长期未练** | 某学员超过7天未完成陪练 | 列表项 + 黄色提醒图标 | 点击一键推送陪练提醒（企业微信/飞书/邮件） |
| **得分骤降** | 某学员单次得分比前3次平均低20分以上 | 临时提醒（弹窗/顶部横幅） | 点击查看该次陪练评估详情 |
| **任务逾期** | 培训任务DDL已过仍未完成 | 列表项 + 橙色警告 | 点击查看逾期任务详情，支持批量催办 |
| **部门异动** | 某部门本月平均分比上月下降超过10分 | 看板顶部横幅提醒 | 点击跳转到该部门对比柱状图 |

**提醒汇总页**：培训经理登录后如有未读异常提醒，概览卡片上方展示"您有N条异常提醒"汇总。所有提醒记录存入操作日志，可追溯管理者的处理动作。

## 4. 数据来源与刷新频率

| 数据源 | 表/字段 | 刷新方式 |
|--------|---------|---------|
| 学员总数 | `users` 表（筛选 org_id + status=active） | 实时查询 |
| 陪练次数 | `training_sessions` 表（筛选 org_id + status=completed + completed_at） | 每小时聚合缓存 |
| 评估得分 | `evaluation_reports` 表（total_score + dimension_scores JSONB） | 每小时聚合缓存 |
| 活跃率 | `training_sessions` + `users` 交叉计算 | 每日更新 |
| 能力雷达图 | `user_progress` 表（dimension_avg JSONB） | 每次陪练结束触发更新 |
| 任务进度 | `task_graph` 表（status + deadline） | 实时查询 |
| 异常提醒 | 定时任务（每日+每周）扫描上述数据表自动生成 | 定期批量计算 |

**缓存策略**：概览卡片和排行榜数据通过Redis缓存（TTL=1小时），减少数据库压力。趋势图长周期数据通过物化视图（每周更新）。

## 5. AARRR指标对应

在B端SaaS场景下，AARRR指标映射如下：

| AARRR阶段 | B端映射 | 对应指标 | 仪表盘体现 |
|-----------|--------|---------|-----------|
| **Acquisition（获客）** | 试用企业数 | 月度新增注册企业 ≥20 | 不直接体现在仪表盘（销售侧数据），但可通过管理后台企业列表统计 |
| **Activation（激活）** | 首次陪练完成 | 注册后7天内完成首次团队陪练的比例 ≥70% | 概览卡片"本周活跃率" + 异常提醒"长期未练" |
| **Retention（留存）** | 月活跃企业数 | 企业年度续费率 ≥85% | 趋势图（12周）+ 活跃率趋势 |
| **Referral（推荐）** | NPS | 培训管理者推荐率（NPS≥60比例）≥50% | 不直接体现（通过定期NPS调研获取），但高活跃率+高得分间接预示高NPS |
| **Revenue（收入）** | ARR | ARR ≥¥500万（Phase 2） | 不直接体现在培训仪表盘（财务侧数据），但陪练次数和活跃企业数是ARR的领先指标 |

**仪表盘核心逻辑**：培训仪表盘重点覆盖 Activation（活跃率/陪练次数）、Retention（趋势图/异常提醒）、间接支撑 Referral（满意度）和 Revenue（使用深度→续费/升级）。获客和收入数据由销售看板和财务系统承载，与培训仪表盘做数据打通（例如在趋势图中标注"本月新增3家付费企业"作为事件标记）。
