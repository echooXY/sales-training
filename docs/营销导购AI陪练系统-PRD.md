# 营销导购陪练智能体（AIDI Studio Training Agent）PRD

> **企业销售团队的AI实战练兵场 — Prompt定义角色，LLM生成战场**

## 1. 产品概述

### 1.1 产品定位

营销导购陪练智能体是一个面向**企业销售团队**的AI实战陪练SaaS平台。它不依赖预标注对话数据，仅凭角色Prompt即可为企业生成逼真的AI顾客，支持文字和语音两种陪练模式。企业培训管理者可在30分钟内完成从场景配置到全团队陪练部署，学员随时发起7×24陪练，管理者通过团队数据看板量化培训ROI。

核心理念来源于 learn-claude-code 工程哲学：不编写决策树或硬编码对话流程，而是为 LLM 提供工具、知识库和上下文管理机制（即 Harness），让模型自主完成角色扮演、对话推进和评估分析。引用技术文档原文：「Agent 不变，Harness 因场景而变。Agent = Model，Harness = Code。模型负责智力，代码负责给模型装工具、管记忆、建团队、做评估。」

**B端差异化价值**：
- **企业培训部门**：从"排课-预约讲师-教室"的线下重模式，转为线上自助式AI陪练
- **销售主管**：从凭经验判断新人能力，转为基于5维雷达图+成长曲线的数据化人才管理
- **企业管理者**：从"培训投入看不清产出"，转为可量化的新人上手周期缩短和团队转化率提升

### 1.2 核心差异化

与现有企业培训方案及竞品的本质区别：

| 维度 | 传统企业培训 | 竞品（Megaview/FindAI） | 本产品 |
|------|-------------|----------------------|--------|
| 部署方式 | 线下讲师+教室 | 云端SaaS | **云端SaaS + 私有化部署可选** |
| 场景创建 | 讲师编写剧本（天级） | 需预标注对话库（周级） | **Prompt即场景（30分钟）** |
| 角色复杂度 | 同事扮演，出戏 | 单一AI顾客 | **多Agent多人决策** |
| 培训覆盖 | 批次式，间隔长 | 7×24但单一难度 | **7×24 + 四级难度体系** |
| 评估方式 | 讲师主观打分 | 关键词匹配 | **独立评估子Agent + 量化5维评分** |
| 管理视角 | 无 | 基础报告 | **团队看板 + 能力雷达图 + ROI追踪** |
| 陪练模式 | 面对面 | 文字 | **文字 + 语音双模式** |
| 数据沉淀 | 纸质/Excel | 云端基础统计 | **每轮对话可归档、可追溯、可审计** |

### 1.3 解决的核心痛点

| 企业痛点 | 传统方案缺陷 | 本产品方案 | 量化预期 |
|---------|-------------|-----------|---------|
| 新人上手慢，影响业绩 | 老员工1对1带教，人均耗时20h+ | AI 7×24陪练，老员工只需抽查评估 | 新人上岗周期从45天→20天 |
| 培训师人效瓶颈 | 1个培训师只能同时带10个新人 | 1个培训师管理100+学员陪练数据 | 人效提升5-10倍 |
| 培训质量不可控 | 不同讲师水平参差，标准不一 | 统一评估量表+mock customer人格一致 | 评估偏差≤5分（百分制） |
| 连锁门店培训难标准化 | 各地门店培训各自为政 | 总部配置场景→一键下发全部门店 | 标准化率100% |
| 培训ROI无法量化 | 只有出勤率，无能力提升数据 | 5维雷达图+成长曲线+转化率归因 | ROI可追踪、可归因 |
| 优秀话术无法沉淀 | 老销售离职带走经验 | 陪练数据归档，金牌话术可提取 | 组织知识资产留存 |

### 3.2 功能详细说明

#### A1 多角色场景陪练（文字模式）

**功能描述**：通过多Agent协作机制，模拟真实销售场景中的多人交互。系统创建1个Lead Agent（教练）和N个Customer Agent（顾客角色），角色之间通过JSONL邮箱异步通信，模拟"真人商量"的效果。

参考来源：[1] 技术文档 第三章（多Agent团队机制）、第四章4.1节（MessageBus通信）

**前置条件**：企业管理员已配置场景模板，学员已登录并选择陪练场景。

**核心机制**：Harness 架构四层叠加

| 机制 | 能力 | 实现方式 |
|------|------|---------|
| s09 Team | 多Agent角色扮演 | spawn_teammate启动独立线程+MessageBus通信 |
| s06 Context | 三层上下文压缩 | microcompact清理旧tool_result、auto_compact超阈值摘要、manual场景切换 |
| s04 Subagent | 子Agent独立评估 | 派生评估Agent，独立messages[]，只读工具 |
| s03+s07 Track | 双层进度追踪 | TodoWrite（单次步骤）+TaskSystem（跨会话DAG） |

**四级难度体系**：

| 难度 | 角色类型 | 行为特征 | 训练重点 |
|------|---------|---------|---------|
| L1 友好型 | 单一顾客 | 性格开朗、容易被说服、会问基本问题 | 基本流程走通 |
| L2 犹豫型 | 单一顾客 | 反复对比、要求更多信息、迟迟不决 | 需求挖掘+价值传递 |
| L3 刁钻型 | 单一顾客 | 价格敏感、质疑产品、提出竞品对比 | 异议处理+竞品应对 |
| L4 多人决策 | 3+角色 | 夫妻/朋友/同事同时在线，互相商量 | 复杂决策沟通+多方协调 |

**主流程**：

1. 学员进入陪练模块，选择「新建陪练」
2. 选择场景模板
3. 系统自动 spawn 角色Agent（Lead Agent + N个Customer Agent）
4. 学员（扮演销售）通过对话界面与AI顾客交互
5. 每轮对话：学员输入→Lead转发→Customer Agent推理→回复→展示
6. 遇到多人场景时，Customer Agent之间自动协商后再回复
7. 学员可随时查看TodoWrite进度
8. 陪练结束，触发评估子Agent生成报告

**异常流程**：
- 角色Agent无响应（30秒超时）：自动重试1次，仍失败则降级为单人模式
- Token超阈值（80K）：触发auto_compact，保存完整对话后压缩为摘要
- 学员中途退出：自动保存当前进度，下次可选择「继续上次陪练」

#### A2 语音陪练模式

**功能描述**：在现有文字陪练基础上，支持实时语音通话模式。特别适用于电话销售（电销）、保险顾问等需要语音沟通能力的岗位培训。学员通过WebRTC与AI顾客进行实时语音对话，对话结束后自动转录存档并以文字模式标准进行评估。

**核心指标**：
- 端到端延迟：≤800ms（ASR 200ms + Agent推理 300ms + TTS 300ms）
- ASR准确率：≥95%（支持中英文混合识别）
- TTS自然度：MOS ≥4.0（支持男/女/老年/年轻四种声线）

**交互模式**：
- PTT模式（Push-to-Talk）：按住说话，松开发送（适合嘈杂环境）
- VAD模式（Voice Activity Detection）：自动检测语音起止（适合安静环境）
- 打断模式：学员可在AI说话时插话，AI在300ms内停止TTS并切换收听

**语音特有功能**：
- 对话结束后自动转录，转录文本与语音录音并行存档
- 语音质量不影响评估（评估子Agent基于转录文本，与文字模式一致）
- 支持逐轮回放（点击对话气泡播放对应语音段）

**降级策略**：
- ASR连续失败2次 → 自动提示切换文字输入
- WebRTC断连 → 3秒内自动重连
- 网络延迟抖动 → 自动降低TTS码率，保证流畅优先

**评估维度扩展**：语音模式评估在5大维度基础上新增「语音表达」子维度（语速/停顿/语调/亲和力），计入沟通表达维度。

#### A3 智能上下文管理

**功能描述**：解决多轮陪练对话中的token膨胀问题，通过三层压缩策略确保30-80轮对话后模型仍保持角色一致性和对话连贯性。语音模式同等受益。

**三层压缩策略**：

| 层级 | 触发条件 | 动作 | 效果 |
|------|---------|------|------|
| microcompact | 每轮LLM调用前 | 清理3轮前的tool_result→[cleared] | 零开销静默清理 |
| auto_compact | 估算token超过阈值 | LLM生成延续性摘要，保存完整记录到.transcripts/ | 关键信息保留+完整归档 |
| manual | 场景切换/主动触发 | 同上，但由Lead Agent决策触发时机 | 按需清理上下文 |

**身份重注入机制**：
压缩后，Lead Agent自动检测到压缩事件，将角色Agent的identity prompt重新注入消息列表，确保压缩后角色人格不丢失。

#### A4 AI评估与反馈

**功能描述**：陪练结束后，独立派生评估子Agent，以只读模式分析完整对话历史，按预设评估量表逐维度打分，生成结构化评估报告。评估子Agent的messages[]在评估完成后丢弃，不污染主Agent上下文。

**多维评分体系**（5大维度 × 16个粒度评分点）：

| 维度 | 权重 | 子评分点 | 说明 |
|------|------|---------|------|
| 沟通表达 | 15% | 开场破冰自然度、语言亲和力、倾听回应 | 基础沟通素养 |
| 需求挖掘 | 25% | 开放式提问使用、需求深挖深度、预算/时间/决策人探测 | SPIN提问法评估 |
| 产品介绍 | 20% | 卖点匹配准确度、FABE转化能力、用户证言使用 | 利益转化能力 |
| 异议处理 | 25% | 认同转化、价值对比、竞品应对、价格谈判 | 核心能力维度 |
| 成交推进 | 15% | 时机把握、紧迫感制造、下一步引导、收尾礼貌 | 成交转化能力 |

**评分标准**：每项0-10分，总分100分。80+优秀，60-80合格，60以下需重练。

**评估子Agent安全边界**：只读工具（read_text/搜索历史对话），禁止写入和修改操作。

#### A5 培训进度管理

**功能描述**：两层进度追踪系统协同工作。

- **TodoWrite（单次陪练）**：每次陪练内部分步骤（破冰→挖需求→推产品→异议处理→逼单→评估），以markdown清单展示，完成后勾选。连续3轮未更新自动注入nag reminder。

- **TaskSystem（跨会话培训计划）**：基于磁盘JSON文件的持久化任务图，支持DAG依赖关系。完成任务后自动解锁blockedBy中的依赖。如「破冰训练」→「需求挖掘」→「FABE介绍」→「价格异议+竞品对比」→「综合实战」。

**能力雷达图**：基于历次评估数据，生成5维度能力雷达图 + 成长曲线，可视化对比每次陪练的进步。

#### B1-B2 场景模板管理与团队管理

**场景模板管理**：
- 培训经理可创建/编辑/发布企业专属场景模板
- 支持从模板库复制现有场景作为起点
- 新建场景保存后立即可用，无需等待数据标注或模型训练
- 支持场景分组管理（按行业/难度/训练目标）
- 支持将场景下发至指定部门/区域/全员

**团队管理**：
- 支持组织架构导入（企业微信/飞书/钉钉同步）
- 支持自定义分组（按区域/门店/入职批次）
- 分级权限：超管/培训经理/区域主管/学员

---

#### B3 培训任务下发

**功能描述**：培训管理者批量创建培训任务，下发至指定团队/个人，设定完成时限，系统自动追踪进度并发送提醒。

- 支持从场景模板库选择任务类型
- 支持批量下发至部门/角色/自定义分组
- DDL到期前24h自动提醒（企业微信/飞书/邮件）
- 逾期未完成自动升级告警至主管
- 支持任务模板（如「新人入职30天培训计划」一键复用）

---

#### B4 团队数据看板

**功能描述**：B端核心价值页面。管理者登录后默认落地页，展示全团队培训数据。

**看板模块**：
1. **概览卡片**：总学员数、本月陪练次数、平均得分、本周活跃率
2. **部门对比**：各区域/门店/小组的陪练次数与平均分对比柱状图
3. **维度短板热力图**：以热力图展示团队在5大维度的整体表现（红色=薄弱）
4. **个人排行榜**：Top10优秀学员 + Bottom10需关注学员
5. **趋势图**：近12周团队平均分变化曲线
6. **异常提醒**：连续3次得分下降/超过7天未陪练的学员自动预警

**权限控制**：
- 销售VP：全公司数据
- 区域主管：本区域数据
- 培训经理：全公司数据（但敏感个人数据脱敏）

#### C3 私有化部署方案

针对金融、保险等强监管行业，提供私有化部署选项。

- 支持Docker Compose / K8s两种部署方式
- LLM可对接企业自有模型（如私有化部署的千问/文心）
- 语音引擎支持离线ASR/TTS（如PaddleSpeech）
- 所有数据不出企业内网

### 6.2 核心数据流

```
学员发送销售话术（文字或经ASR转录）
    ↓
Lead Agent 主循环：
    1. microcompact（清理旧tool_result）
    2. 检查token阈值 → 必要时auto_compact
    3. 读取MessageBus收件箱（Customer Agent回复）
    4. LLM推理（基于System Prompt + 对话历史 + inbox消息）
    5. 执行tool调用（TodoWrite更新进度 / spawn新角色 / evaluate评估）
    6. 将Lead回复写入MessageBus发件箱 → 推送给Customer Agent
    ↓
Customer Agent 独立线程：
    1. 读取MessageBus收件箱（Lead发来的销售话术 + 其他角色商量内容）
    2. LLM推理（基于角色Prompt + inbox消息）
    3. 生成回复写入MessageBus
    ↓
前端展示（AI顾客回复 → 文字展示 / TTS语音播放）
```

## 12. 风险评估

| 风险 | 概率 | 影响 | 缓解措施 |
